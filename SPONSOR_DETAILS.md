## About Cursor

Cursor is an AI-first code editor designed to support developers across the spectrum of AI-assisted programming, from autocomplete to autonomous agents. This guide introduces the fundamentals and walks through key features to help you get productive quickly.

### What is Cursor?

Cursor is a programming environment powered by large language models (LLMs). It lets you generate, edit, and reason about code with AI. Cursor is model-agnostic and includes tools for both light-touch and full-agent workflows.

**Key concepts:**

- **LLMs**: Trained to predict the next token in a sequence. They operate using "context," which includes the current conversation, attached files, and system prompts.
- **Prompt**: The instruction or input you give the model.
- **Context**: The data the model sees when generating a response.
- **Tokens**: Units of text that determine the size and cost of input/output.
- **Model selection**: Choose from different LLMs based on reasoning strength, speed, or cost.

### Core Features

These tools help you work with AI at varying levels of autonomy.

#### Tab (Autocomplete)

- In-house autocomplete model trained on code.
- Suggests multiple lines or blocks.
- Navigates within and across files.
- Stays close to how developers write code manually.

#### Inline Edit

- Highlight code and ask for edits.
- Great for refactoring, rewriting, or adding documentation.
- Triggered with Cmd+K after selecting code.

#### Agent

- AI agent with read/write access to your codebase.
- Modes: Ask (responds to queries) and Agent (takes action).
- Context menu integration for file-level tasks.
- Agent Loop allows multi-step reasoning and tool use.
- Supports background execution on remote VMs.
